⚠ Pourquoi les tables ne s'affichent pas ?
Supabase bloque la lecture du schéma via anon. Choisissez l'une de ces deux solutions :
Solution 1 — Créer une fonction SQL (recommandé)

Dans supabase.com → SQL Editor, exécutez ce code une seule fois :

create or replace function get_tables() returns table(table_name text) language sql security definer set search_path = public as $$ select table_name::text from information_schema.tables where table_schema = 'public' and table_type = 'BASE TABLE' order by table_name; $$; grant execute on function get_tables() to anon, authenticated;
1. Ouvrez votre projet Supabase → SQL Editor
2. Collez et exécutez le code ci-dessus
3. Reconnectez-vous ici avec votre clé anon
ou
Solution 2 — Clé service_role

Dans Settings → API, copiez la clé service_role.
⚠ Ne l'exposez jamais côté client en production.
